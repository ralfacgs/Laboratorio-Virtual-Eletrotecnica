Notas de Engenharia: Módulo de Potência e Defasagem

Este documento detalha os modelos matemáticos e de programação implementados no arquivo modulo2_potencia_e_defasagem.html.

1. Modelagem das Equações Instantâneas

As formas de onda plotadas no osciloscópio digital seguem a modelagem matemática convencional para circuitos de Corrente Alternada. A tensão senoidal pura serve como a referência angular do sistema (0°). A corrente elétrica é modelada com base no atraso ou adiantamento provocado pela reatância da carga (φ).

// Tensão instantânea: v(t) = Vp * sin(ωt)

const v = Vp * Math.sin(omega * tSec - phaseShift);

// Corrente instantânea defasada: i(t) = Ip * sin(ωt - φ)

const i_val = Ip * Math.sin(omega * tSec - phiRad - phaseShift);

2. Potência Instantânea p(t) e Frequência Dupla

A curva de potência instantânea é obtida através do produto ponto a ponto dos vetores de tensão e corrente: p(t) = v(t) · i(t). Matematicamente, a expansão trigonométrica desse produto comprova o fenômeno da frequência dupla observado na tela do osciloscópio:

p(t) = Vrms · Irms · cos(φ) - Vrms · Irms · cos(2ωt - φ)

A presença do termo 2ω explica pedagogicamente por que a onda de potência (cor de âmbar) oscila a 120 Hz enquanto as fontes de sinal operam em 60 Hz. Quando a curva cruza a linha de zero e entra na região negativa, o simulador evidencia a energia sendo devolvida pelos campos magnéticos/elétricos da carga de volta para a rede.

3. Cálculos do Triângulo de Potência

Para preencher os blocos de métricas, o script realiza a conversão dos valores de pico informados nos seletores numéricos para valores eficazes (Root Mean Square), aplicando as leis fundamentais do triângulo de potência:

const Vrms = Vp / Math.sqrt(2);

const Irms = Ip / Math.sqrt(2);

const S = Vrms * Irms; // Potência Aparente [VA]

const P = S * Math.cos(phiRad); // Potência Ativa [W]

const Q = S * Math.sin(phiRad); // Potência Reativa [VAr]

const FP = Math.cos(phiRad); // Fator de Potência

4. Base de Tempo e Proteção Anti-Aliasing

O simulador incorpora um controle de zoom equivalente ao ajuste de "Time/Div" físico, permitindo a análise de 0.5 a 1.000 ciclos. Para evitar que altas frequências ou muitos ciclos descaracterizem o desenho da onda (subamostragem), foi implementado um motor de amostragem dinâmica que calcula os pontos ideais com um teto protetor de memória de 5.000 amostras simultâneas.

5. Sincronismo de Fase e Animação Temporal

O fluxo de tempo (requestAnimationFrame) foi projetado para responder fidedignamente à frequência. O incremento da variável phaseShift foi amarrado ao parâmetro f (frequência). Dessa forma, se o aluno dobrar a frequência de 60 Hz para 120 Hz durante a animação ativa, a velocidade de varredura visual das ondas na tela também dobrará proporcionalmente.
