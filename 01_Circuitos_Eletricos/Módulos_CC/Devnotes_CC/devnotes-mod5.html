Este documento detalha a arquitetura matemática e os algoritmos gráficos implementados no arquivo modulo_cc_5_simulador_cc.html.

1. Motor de Cálculo (Modified Nodal Analysis - MNA)

A resolução do circuito não obedece a rotinas fixas (como divisores de tensão pré-programados), mas sim a um motor de Análise Nodal Modificada. O algoritmo constrói dinamicamente a matriz de condutâncias ($A$) e o vetor de correntes/tensões conhecidas ($Z$), resolvendo o sistema linear bidimensional $A \cdot X = Z$ através da decomposição LU, utilizando a biblioteca externa math.js.

// Alocação de componentes na Matriz de Condutância Estática

const g = 1 / res.value;

if (i !== -1) A[i][i] += g; // Diagonal principal

if (j !== -1) A[j][j] += g;

if (i !== -1 && j !== -1) { A[i][j] -= g; A[j][i] -= g; }

2. Câmera Virtual e Interação Espacial

A área gráfica foi modelada como um plano infinito utilizando a API nativa do HTML5 Canvas. A manipulação de visualização (Pan & Zoom) utiliza matrizes de transformação de estado em tempo real (translate e scale). O algoritmo de zoom calcula o deslocamento relativo do mouse para garantir que o escalonamento ocorra exatamente sob o cursor, otimizando a UX (User Experience).

// Cálculo do offset de Câmera sob o ponteiro

const zoomDirection = e.deltaY > 0 ? -1 : 1;

let newScale = scale * (zoomDirection > 0 ? 1.1 : 1 / 1.1);

offsetX = mouseX - worldX * newScale;

offsetY = mouseY - worldY * newScale;

3. Algoritmo de Roteamento Ortogonal Automático

Para garantir o padrão visual esquemático de engenharia, os ramos de conexão nunca formam retas diagonais entre dois nós. O sistema calcula o perímetro de ligação entre as coordenadas de origem e destino, determinando o caminho mais curto usando dobras ortogonais ($90^{\circ}$) em formato |- ou -| para os fios e terminais de componentes.
