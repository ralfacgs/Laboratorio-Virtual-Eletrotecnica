Notas de Engenharia: Leis de Kirchhoff

Este documento detalha a arquitetura matemática base e os modelos de malhas implementados na engine de simulação.

1. Resolução Lógica por Topologia Circuital

O núcleo do simulador gerencia dinamicamente o cálculo da resistência equivalente (req) com base na propriedade circuitoAtual. O motor do sistema suporta circuitos puramente em série, paralelos ou arranjos mistos complexos de até 4 resistores ôhmicos.

// Topologias Série (Série Simples e 3 Resistores)

req = r1 + r2 + (numR === 3 ? r3 : 0);

it = v / req;

// Topologias Paralelas (Paralelo Simples e 3 Ramos)

let invReq = (1/r1) + (1/r2) + (numR === 3 ? 1/r3 : 0);

req = 1 / invReq;

2. Formulação de Malhas Mistas (Análise de Nós Internos)

Para as topologias mistas (mista e mista2), o script realiza uma redução em etapas. Primeiro, calcula-se o equivalente paralelo dos resistores internos (R₂ e R₃). Posteriormente, as quedas de tensão na malha principal são subtraídas para determinar o potencial efetivo nos nós de ramificação (va).

const rp = (r2 * r3) / (r2 + r3);

req = r1 + rp + (circuitoAtual === 'mista2' ? r4 : 0);

it = v / req;

v1 = r1 * it;

v4 = (circuitoAtual === 'mista2') ? (r4 * it) : 0;

let va = v - v1 - v4; // Tensão resultante no nó compartilhado

3. Validação das Leis de Kirchhoff (LKC e LKT)

O painel expansível de expressões matemáticas (equationBox) atua como um validador pedagógico em tempo real das leis fundamentais:

  • Lei dos Nós (LKC): Demonstra graficamente que a corrente total da malha converge exatamente com a soma das correntes derivadas dos sub-ramos (It = I₂ + I₃).
  • Lei das Malhas (LKT): Comprova que a soma algébrica das diferenças de potencial ao longo do percurso fechado é nula (Vfonte - VR₁ - Vparalelo - VR₄ = 0).

4. Dissipação Térmica Dinâmica (Efeito Joule Visual)

A função drawUniversalResistor monitora a taxa de potência convertida em calor (P = V · I). Caso a potência calculada ultrapasse os limiares críticos estabelecidos na física do ambiente virtual, o gradiente térmico do componente é alterado dinamicamente via Canvas API:

let color = overrideColor || "#334155"; // Estado nominal (cinza)

if (power > 0.5) color = "#b45309"; // Aquecimento leve (marrom)

if (power > 2.0) color = "#ea580c"; // Estresse térmico (laranja)

if (power > 5.0) color = "#ef4444"; // Dissipação crítica (vermelho brilhante)

// Ativação do efeito glow em alta potência

if (power > 1.0) { ctx.shadowBlur = 12; ctx.shadowColor = color; }

5. Cinemática de Partículas e Mapeamento Geométrico

A representação cinemática do fluxo elétrico é gerenciada de forma assíncrona. Os elétrons (drawPathDots) avançam sob incrementos de offsets dic_circuitos_eletricosados à intensidade de corrente local, normalizados pela constante linear SPEED = 15. A rotina getResistorOffset injeta uma matriz de deformação geométrica no vetor de translação, forçando os pontos a rastrearem fielmente o perfil em zigue-zague dos condutores internos.
