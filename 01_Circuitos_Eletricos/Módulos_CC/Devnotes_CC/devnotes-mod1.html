Notas de Engenharia: Lei de Ohm e Potência

Este documento detalha a arquitetura matemática base implementada no arquivo modulo_cc_1_lei_ohm.html.

1. Modelagem Matemática Básica (Leis de Ohm e Joule)

O núcleo do simulador fundamenta-se nas equações primárias da eletrodinâmica. Em cada frame de animação, o algoritmo capta os valores dos sliders de Tensão ($V$) e Resistência ($R$) e computa a Corrente Elétrica ($I$) e a Potência Dissipada ($P$) em tempo real.

// Primeira Lei de Ohm (Cálculo de Corrente)

const i = v / r;

const i_ma = i * 1000; // Conversão visual para miliAmpères

// Lei de Joule (Cálculo de Potência Ativa)

const p = v * i;

2. Limites Físicos da Interface (Validação de Inputs)

Para garantir a estabilidade do motor matemático, os elementos `input type="range"` possuem barreiras estritas. A resistência mínima foi travada em $10\Omega$ e a máxima em $1000\Omega$. Isso previne o erro crítico de "divisão por zero" na fórmula $I = V / R$ caso o usuário zerasse a carga, o que causaria uma corrente infinita ($I = \infty$) e travaria o laço de animação do Canvas.

3. Motor Elétron-Cinético (Velocidade Dinâmica)

A ilusão de fluxo contínuo de elétrons é gerada manipulando uma variável de progresso (animationProgress). O passo (speed) de deslocamento das partículas a cada ciclo da função requestAnimationFrame é diretamente proporcional ao valor calculado da Corrente ($I$).

// A velocidade é um multiplicador da corrente real

let speed = (i * 20);

// Teto de segurança para evitar saltos (stuttering) visual a 60 FPS

if (speed > 5) speed = 5;

animationProgress = (animationProgress + speed) % perimeterLength;

4. Simulação Termográfica (Dissipação Térmica)

A representação de estresse térmico no componente resistor utiliza injeções dinâmicas na API do Canvas. Conforme a Potência ($P$) atinge limiares predefinidos (> 0.5W, > 2W, > 5W), a cor do traçado (strokeStyle) muda de um cinza passivo para vermelho intenso, acionando também a propriedade shadowBlur para emular uma coroa de radiação térmica (efeito glow).

let color = "#334155"; // Estado Frio (Cinza)

if (power > 0.5) color = "#b45309"; // Aquecendo (Marrom/Âmbar)

if (power > 2.0) color = "#ea580c"; // Quente (Laranja)

if (power > 5.0) color = "#ef4444"; // Sobrecarga (Vermelho Brilhante)

5. Simbologia e Geometria Vetorial

A simbologia do resistor não é uma imagem estática, mas sim um vetor desenhado puramente através da matemática (linhas 2D), seguindo a recomendação gráfica do padrão normativo norte-americano (ANSI). Além disso, implementamos um sistema de interpolação geométrica que obriga as partículas de fluxo de carga a acompanharem o formato exato do "zig-zag", ao invés de cortarem o desenho por uma reta ilusória.
